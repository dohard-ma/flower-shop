# Processing Rules (处理规则)

你需要对输入 JSON 中的以下 4 个字段进行强制清洗：

1. **materials (数组)**
    * **目标格式：** `["标准主花材名称"]` (数组中仅包含一项)。
    * **逻辑：** 优先识别原 `materials[0]`，若为空，则根据 `name` (商品名) 进行语义匹配。
    * **规则：** 必须严格匹配【标准主花材列表】中的一项。若无法识别，默认填入 `["混搭花束"]` 或 `["其他主花材"]`。若遇到“零食”、“玩偶”等，映射为 `["其他主花材"]`。

2. **colorSeries (字符串)**
    * **目标格式：** 字符串，必须属于【标准色系列表】。
    * **逻辑：** 优先保留原 `colorSeries`（若已合规）。若为空或不合规，根据 `name` 和 `description` 提取颜色。
    * **规则：** 若无法确定，默认填入 `"混搭色"`。

3. **style (字符串)**
    * **目标格式：** 字符串，必须属于【标准款式列表】。
    * **逻辑：** 将原 `style` 或 `category` 映射到标准列表。
    * **特殊映射：** 若原值为 "花盒"，必须映射为 `"礼盒"`。

4. **targetAudience (数组) - ⚠️重点规则**
    * **目标格式：** `["人群A", "人群B", "人群C"]` (数组长度必须严格等于 3)。
    * **逻辑：** 优先使用原 `targetAudience`。若原数据不足 3 个，或包含非【标准适用人群列表】中的选项，必须根据 `name` 的语境和常识从【标准适用人群列表】中进行智能补全。
    * **补全策略示例：**
        * 送“女友”，可补全为 `["女友/老婆", "闺蜜", "朋友/同事"]`。
        * 送“长辈”，可补全为 `["长辈", "父母", "老师"]`。
        * 送“儿童”，可补全为 `["儿童", "学生", "新生儿"]`。
